# NexusAI —— 工业级双核智能知识引擎 (v3.1)

NexusAI 是一款基于 **Spring Boot 3**、**LangChain4j** 和 **RAG** 架构的生产级 AI 后端系统。系统通过“分布式记忆”与“动态知识摄取”双轮驱动，实现了具备实时学习能力与长效会话感知的智能问答体验。

## 🌟 v3.1 核心特性
- **分布式会话记忆 (Distributed Memory)**：集成 **Redis** 自研 `RedisChatMemoryStore`，支持跨服务重启的会话持久化，完美实现多轮对话中的指代消解。
- **动态知识热加载 (Hot-loading Ingestion)**：开发 `/ai/upload` 实时摄取接口，支持前端 PDF 动态上传并即时完成向量化索引，无需重启服务即可更新 AI 知识边界。
- **双核心智力入口 (Dual-Core Architecture)**：
    - **专业知识库 (Professional RAG)**：基于 ChromaDB 持久化存储与纳米级切片策略，提供零幻觉的精准检索。
    - **通用助手 (General Intelligence)**：注入 2026 年时空感知补丁，处理高灵活性逻辑咨询。
- **打字机交互 UI 2.0**：对标 Apple / 影视飓风审美，优化 1100px 响应式布局，集成 **Smooth Auto-scrolling** 平滑追踪算法。

## 🛠️ 技术栈（企业标准版）
- **后端核心**: Java 21 (LTS), Spring Boot 3.2.5
- **AI 编排**: LangChain4j 0.34.0, 智谱 AI (GLM-4-Flash)
- **存储中心**: **ChromaDB 0.5.20** (向量持久化), **Redis** (会话持久化)
- **文档处理**: Apache Tika (高性能解析)
- **交互协议**: SSE (Server-Sent Events) 异步长连接

## 📈 版本深度演进历程 (Version Timeline)

### v3.1 (当前版本) - 动态摄取与稳定性加固
- **[3.1.1] 动态热加载**：实现基于 `MultipartFile` 的文件摄取链路，支持知识库零停机更新。
- **[3.1.2] 健壮性优化**：补全 `TokenStream` 的异常捕获逻辑，修复了流式传输中常见的配置校验报错。
- **[3.1.3] 布局重构**：修正前端容器塌陷 Bug，将对话框升级为 1100px 工业级沉浸式布局。

### v3.0 - 分布式会话闭环
- **[3.0.1] Redis 存储集成**：通过自定义序列化器将对话快照同步至 Redis，解决 JVM 重启失忆痛点。
- **[3.0.2] 会话逻辑隔离**：引入 `@MemoryId` 机制，确保多用户并发场景下的对话上下文精准隔离。

### v2.4 - UI 联动与时空同步
- **[2.4.1] 双大脑联调**：构建「极光紫」与「苹果蓝」双端 UI，实现产品级功能闭环。
- **[2.4.2] 时空补丁**：注入 SystemMessage 纠正模型时效性幻觉，同步 2026 年时间线。

### v2.3 - 数据治理与检索调优
- **[2.3.1] 纳米级切片**：采用 `Micro-Chunking` (100字/片) 策略，从物理层解决章节语义干扰。
- **[2.3.2] 检索降噪**：固定 Top-1 召回阈值，实现生产级回答一致性。

### v2.0 - 存储持久化转型
- **[2.0.1] 容器化部署**：锁定 ChromaDB v0.5.20 镜像，实现数据卷物理挂载，保护系统盘空间。

### v1.0 - RAG 链路原型点火
- 基于 Spring Boot 3 对接 LLM API，打通 PDF 加载与向量检索基础链路。

## 🚀 快速启动
1. **基础设施**: 启动 Docker `my-chroma` 容器与本地 Redis Server。
2. **环境配置**: 操作系统环境变量中设置 `ZHIPU_API_KEY`。
3. **数据接入**:
    - 方式 A：直接存入 `/documents` 文件夹并启动。
    - 方式 B：通过 `index.html` 界面点击“📁 喂入新知识”动态上传。
